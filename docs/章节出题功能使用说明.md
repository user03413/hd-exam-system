# 章节出题功能使用说明

> **功能上线时间**: 2026-03-25  
> **适用对象**: 教师用户

---

## 📖 功能概述

教师可以针对题库中的某一章节生成专属考试链接，学生通过该链接参加考试时，只能抽到该章节的题目。

---

## 🎯 使用场景

### 场景1: 单元测验
教师讲解完某个章节后，可以针对该章节生成考试链接，进行单元测验。

### 场景2: 专题练习
针对学生薄弱的章节，生成专门的练习链接。

### 场景3: 分阶段考核
按照教学进度，逐步开放不同章节的考试。

## 👨‍🎓 学生端体验

### 访问链接

学生点击链接后：

1. 页面顶部显示：**专题考试：第一章**
2. 输入学号验证身份
3. 页面显示：**欢迎，XXX 同学！（第一章专题考试）**
4. 开始答题，题目全部来自第一章

### 考试规则

- **题目数量**: 10题（4单选 + 3多选 + 3简答）
- **分值**: 每题10分，满分100分
- **题目来源**: 仅限所选章节
- **随机抽题**: 每次考试题目随机抽取

## 🔧 技术实现

### 后端API

#### 1. 获取章节列表
```http
GET /api/exam/chapters
```

**响应示例**：
```json
{
  "success": true,
  "chapters": [
    {"chapter": "第一章", "count": 35},
    {"chapter": "第二章", "count": 28}
  ]
}
```

#### 2. 生成章节考试链接
```http
GET /api/exam/chapter-link?chapter=第一章
```

**响应示例**：
```json
{
  "success": true,
  "link": "https://hd-exam-api.771794850.workers.dev/exam?chapter=第一章&linkId=abc123",
  "chapter": "第一章",
  "linkId": "abc123"
}
```

#### 3. 学生验证（带章节）
```http
POST /api/exam/verify
Content-Type: application/json

{
  "student_id": "220252216068",
  "chapter": "第一章"
}
```

#### 4. 开始考试（按章节抽题）
```http
POST /api/exam/start
Content-Type: application/json

{
  "session_id": "xxx"
}
```

系统会根据session中保存的chapter信息，只从该章节抽题。

## ⚠️ 注意事项

### 1. 章节题目数量

确保所选章节有足够的题目：
- 单选题至少4题
- 多选题至少3题
- 简答题至少3题

如果题目不足，系统会提示错误。

### 2. 链接有效期

- 链接永久有效
- 学生可以重复考试
- 每次考试题目随机抽取

### 3. 考试记录

- 每次考试的章节信息会被记录
- 可以区分是专题考试还是全题库考试
- 方便后续统计分析

## 🐛 常见问题

### Q1: 生成的链接学生无法访问？
**A**: 检查网络连接，确保学生可以访问 Cloudflare Workers 地址。

### Q2: 学生考试时题目不是所选章节？
**A**: 确保学生使用的是正确的章节链接，而不是普通的考试入口。

### Q3: 某个章节题目不够怎么办？
**A**: 该章节题目不足时，建议选择"全题库"模式，或者合并多个章节。

### Q4: 可以同时生成多个章节的链接吗？
**A**: 可以！每次生成一个链接，可以生成多个不同章节的链接。

### Q5: 学生可以看到是哪个章节的考试吗？
**A**: 可以！页面会明确显示"专题考试：第X章"。
