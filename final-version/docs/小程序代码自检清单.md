    // ...

### 2. utils/request.js
**改动内容：**
- 移除顶层 `const app = getApp()` 调用
- 在函数内部调用 `getApp()`
- 添加 try-catch 错误处理
- 添加详细的日志输出
- 定义 DEFAULT_BASE_URL 常量作为默认值

**关键代码：**
```javascript
const DEFAULT_BASE_URL = 'https://90c19216-7224-4e07-9c9b-2d1be18d1149.dev.coze.site'

function getBaseUrl() {
  try {
    const app = getApp()
    if (app && app.globalData && app.globalData.baseUrl) {
      return app.globalData.baseUrl
    }
  } catch (err) {
    console.warn('[Request] 获取 globalData.baseUrl 失败，使用默认值')
  }
  return DEFAULT_BASE_URL
}
```

### 3. 页面文件
**改动内容：**
- 所有 `getApp()` 调用都在生命周期函数或事件处理函数内
- 添加 `try-catch` 错误处理
- 添加 `app && app.globalData` 存在性检查
- 添加详细的日志输出
- 添加防止重复提交的 loading 状态检查

**关键代码模式：**
```javascript
// 正确的 getApp() 调用方式
onLoad() {
  try {
    const app = getApp()
    if (app && app.globalData) {
      // 使用 app.globalData
    }
  } catch (err) {
    console.error('获取 App 失败:', err)
  }
}
```

## ⚠️ 开发环境配置

### 必须关闭域名校验

在微信开发者工具中：
```
1. 点击右上角「详情」按钮
2. 选择「本地设置」标签
3. 勾选「不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书」
4. 重新编译
```

## 🎯 问题排查

如果仍然出现错误，请检查：

1. **Console 错误信息**
   - 打开调试器 Console 标签
   - 查看红色错误信息

2. **Network 请求状态**
   - 打开调试器 Network 标签
   - 查看请求是否发送成功

3. **确认域名校验已关闭**
   - 详情 → 本地设置 → 勾选不校验域名

4. **确认已清除缓存并重新编译**
